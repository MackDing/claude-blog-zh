但我们的网络安全工作不仅限于 Glasswing：通过 Claude Security，更广泛的组织可以在其代码库中使用我们最强大的通用模型 Claude Opus 4.7。Opus 4.7 是目前可用于发现和修补软件漏洞以及发现可能被忽略的复杂、依赖上下文问题的最强模型之一。

Claude Security——以前称为 Claude Code Security——已经在有限的研究预览中接受了数百个各种规模组织的测试，帮助团队扫描其代码库中的漏洞并生成针对性的补丁。他们的反馈塑造了今天的版本，使 Claude Security 向所有企业版客户开放。它配备了定时和针对性扫描、更轻松的审计系统集成以及改进的已分类发现跟踪。无需 API 集成或自定义 agent 构建：如果您的组织使用 Claude，您今天就可以开始扫描。

Opus 4.7 的功能还通过 Claude 集成到许多企业已经使用的软件工具中，为网络防御者提供服务。我们的技术合作伙伴，包括 CrowdStrike、Microsoft Security、Palo Alto Networks、SentinelOne、TrendAI 和 Wiz 正在将 Opus 4.7 嵌入到他们的工具中；此外，Accenture、BCG、Deloitte、Infosys 和 PwC 等服务合作伙伴现在正在帮助组织部署集成了 Claude 的安全解决方案。

我们正在进入网络安全的关键时期。AI 正在压缩从漏洞发现到利用之间的时间线。我们认为正确的应对方式是确保防御者能够以最便捷的方式访问前沿能力，无论是直接通过 Claude 还是通过我们的合作伙伴。

## Claude Security 的工作原理

Claude Security 可以直接从 Claude.ai 侧边栏访问，或访问 claude.ai/security。首先，选择您的一个代码仓库（或将范围限定到特定目录或分支），然后开始扫描。

在扫描时，Claude 像安全研究人员一样对代码进行推理。Claude 不是通过搜索已知模式来发现漏洞，而是寻求理解组件如何跨文件和模块交互，跟踪数据流并阅读源代码。

扫描完成后，Claude 会对其每项发现提供详细说明，包括漏洞真实性的置信度、严重程度、可能的影响以及如何重现。它还会生成针对性补丁的说明，用户可以在 Web 版 Claude Code 中打开以在上下文中完成修复。

## 我们从初始预览中学到的经验

在过去两个月中，我们根据数百家企业在生产环境中使用 Claude Security 的经验对其进行了改进。具体来说，我们发现：

检测质量至关重要。团队告诉我们，高置信度的发现才是真正加速安全工作的关键。Claude Security 的多阶段验证流程会在每项发现到达分析师之前独立检查它，这降低了误报率，并且 Claude 会为每个结果附上置信度评级。这意味着团队收到的信号值得采取行动。

从扫描到修复的时间是最重要的指标。早期用户一致指出这一点，有几个团队在一次会议中就完成了从扫描到应用补丁的过程，而不是在安全和工程团队之间来回沟通数天。

团队希望持续覆盖，而不是一次性审计。我们添加了定时扫描选项，以便团队可以设置定期审查和处理发现的节奏。

在此版本中，我们还添加了针对代码仓库内特定目录的扫描能力，可以记录原因并关闭发现（以便未来的审查者可以信任先前的分类决策），将发现导出为 CSV 或 Markdown 格式以用于现有的跟踪和审计系统，以及通过 webhook 将扫描结果发送到 Slack、Jira 或其他工具。
